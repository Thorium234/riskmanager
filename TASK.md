# TASK.md

## Objective

Build a professional, high-conversion landing page for the Risk Intelligence Register (RIR).

This is NOT just a UI task.
It must clearly communicate value, structure the product, and prepare for SaaS scaling.

---

## Landing Page Sections (Strict)

### 1. Hero Section

* Headline:
  "Identify, Predict, and Mitigate Risks Before They Happen"
* Subtext:
  "AI-powered risk intelligence built for developers"
* CTA Buttons:

  * Get Started
  * View Templates

---

### 2. Problem Section

Highlight:

* Developers relying on unstructured tools
* Lack of standard risk tracking
* Excel not suitable for technical workflows

---

### 3. Solution Section

3 pillars:

1. Structured Risk Registers
2. AI Risk Intelligence
3. Developer-First Workflow

---

### 4. Features Section

* Prebuilt templates
* Risk scoring
* AI suggestions
* Clean UI (no spreadsheet feel)

---

### 5. How It Works

Step 1: Choose template
Step 2: Add risks
Step 3: AI analysis

---

### 6. Use Cases

* Developers
* Startups
* Technical teams

---

### 7. Final CTA

"Stop reacting to risks. Start predicting them."

---

## Design Requirements

* Clean, modern, minimal
* Dark mode preferred
* Developer aesthetic (similar to Vercel, Linear)
* Responsive design

---

## Technical Requirements

* Use Next.js App Router
* Use Tailwind CSS
* Component-based structure
* Reusable UI components

---

## Folder Expectations

* /components → reusable UI
* /app → page structure
* /styles → global styles

---

## Gravity Prompt

Use this prompt inside Gravity:

---

You are a senior Next.js engineer.

Build a high-conversion landing page for a product called "Risk Intelligence Register (RIR)".

Requirements:

* Use Next.js (App Router)
* Use Tailwind CSS
* Create a clean, modern developer-focused UI
* Dark mode design
* Fully responsive

Page must include:

1. Hero section with headline, subtext, and CTA
2. Problem section explaining why Excel and unstructured tools fail developers
3. Solution section with 3 pillars (structured registers, AI intelligence, developer workflow)
4. Features section
5. How it works (3 steps)
6. Use cases
7. Final CTA

Constraints:

* Do not use generic placeholder text
* Use strong, clear product messaging
* Avoid corporate buzzwords
* Make it feel like a real SaaS product (similar quality to Vercel or Linear)

Output:

* Full page code
* Reusable components
* Clean folder structure

---

## Execution Rule

Do not move to backend or AI until:

* Landing page is complete
* Messaging is clear
* UI looks production-ready

---

## Final Note

If the landing page does not clearly explain:

* What the product is
* Who it is for
* Why it matters

Then it has failed.
